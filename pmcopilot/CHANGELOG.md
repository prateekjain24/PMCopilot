# Changelog

## 1.0.0 (2026-03-22)

Initial release of PMCopilot.

### Skills (12)
- prd-generator: PRD generation with Google, Amazon PRFAQ, and Stripe templates
- prioritize: Feature prioritization with RICE, ICE, MoSCoW, Kano, Weighted Scoring, Cost of Delay
- roadmap: Roadmap planning in Now/Next/Later or outcome-based formats
- sprint-review: Sprint analytics with Jira integration and velocity trends
- competitive-teardown: Full competitive analysis with parallel agent orchestration
- market-sizing: TAM/SAM/SOM estimation with web research
- user-research: Personas, interview guides, JTBD canvases, transcript analysis
- experiment-design: A/B test design with sample size calculations
- stakeholder-update: Weekly, monthly, and executive summary updates
- app-store-intel: App Store and Play Store intelligence
- launch-checklist: Launch readiness checklists (soft/hard/beta)
- metrics-review: North Star Metric and AARRR Pirate Metrics analysis

### Agents (7)
- prd-writer: Write PRDs using best-practice templates
- sprint-analyst: Analyze Jira/Linear sprint data
- data-analyst: Query Amplitude/Mixpanel for product metrics
- research-synthesizer: Orchestrate other agents, synthesize multi-source findings
- app-teardown: Navigate iOS/Android apps on simulators, screenshot every screen
- web-teardown: Browse competitor websites via Chrome automation
- ux-reviewer: Review screenshots for UX quality against heuristics

### Custom MCP Servers (4)
- pm-frameworks (12 tools): RICE, ICE, Kano, MoSCoW, Weighted Scoring, Opportunity Scoring, Cost of Delay (CD3), TAM/SAM/SOM, sample size calculator, significance test
- simulator-bridge (15 tools): iOS Simulator control via xcrun simctl and idb
- emulator-bridge (16 tools): Android Emulator control via adb
- app-store-intel (10 tools): App Store and Play Store data extraction

### Integrations
- Jira/Confluence: Sprint data, ticket management, Confluence publishing
- Slack: Post updates, search discussions, create canvases
- Amplitude/Mixpanel: Product analytics queries (optional)
- Figma: Design file references (optional)
- Granola: Meeting transcripts for user research analysis
- Gmail: Draft stakeholder emails, search feedback
- Google Calendar: Meeting context, scheduling
